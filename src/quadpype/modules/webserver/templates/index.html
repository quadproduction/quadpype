What is it?

The QuadPype Webserver API, serves as a seamless bridge between QuadPype software and external systems, enabling developers to interact with published production data effortlessly.
Designed for performance and scalability, this API provides a robust interface for retrieving critical information about production workflows, ensuring an efficient and streamlined integration process.

Key Features

Beyond simple data retrieval, the Webserver API opens a world of possibilities to enhance studio operations. Its powerful features include:

  • Studio Settings Management

    Easily update and configure your studio settings for optimal production workflows.

  • Project Oversight

    Manage and organize the projects with external tools that can send requests to update configurations.

  • User Administration

    Browse and manage the user list efficiently, ensuring streamlined access and role assignments.

  • User Notifications

    Notify connected users in real-time through intuitive pop-ups or tray notifications, keeping everyone informed and aligned.

Extensibility

The Webserver API's architecture is built with flexibility in mind. With minimal development effort, new features and functionalities can be added by defining additional API routes that connect directly to the QuadPype core. This extensibility ensures that the API evolves with the unique demands of your studio environment.

Open Source Collaboration

Like the QuadPype core, the Webserver API is maintained by the Quad Dev Team as an open-source project. We actively encourage contributions to enhance and expand the API's capabilities. For more information on how to contribute, visit the project's wiki page and join the vibrant community helping to shape the future of this powerful tool.
